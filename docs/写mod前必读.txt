# ！！！重要，开搞之前请先看一下注意事项！！！
# 1.给你的mod想一个代号，这个代号一定要特殊，最好根据mod的主要内容来确定。
#   mod的代号不要叫submod或者extension，没有特殊性，有导致mod冲突的风险。
#   mod文件的命名格式  gts_mod代号_xxxx
#   mod内事件，flag等变量的命名格式  gts_mod代号_xxxx

# 2.做mod的时候可以修改原版的文本或者贴图，但是尽量不要改原版的代码逻辑

# 3.mod发布前，记得写一个说明文件介绍一下你的mod增加了什么内容
#   说明内容尽量全面一些，方便玩家体验新内容

# ！！！文件命名格式！！！
# 任何文件:避免mod冲突，一律以gts_为前缀，gts_后面可以跟作者名/项目名，加载顺序问题可以再前面加0或者z：000_gts_

# common:如果你的项目包含大量新内容，建议新建一个txt，替换原版内容*必须*单独放在新的txt中，并加上_replace后缀

# events:替换原版事件*必须*单独放在新的txt中，并加上_replace后缀

# gfx/interface/sound:不要重名即可

# localisation: 你的mod围绕什么展开就用什么前缀，比如起源、领袖、国策、事件链
# gts_civ（国策） gts_ap(遗珍) 如果你要围绕一个法令写一堆文案的话可以再开个gts_edict
# gts_o(起源)>gts_i(领袖)>gts_evt(事件链)>gts_com(其他) 前者优先于后者，起源里有领袖的话，文件应该命名为gts_o
# 原版本地化文本中的 :0 一类的数字是给它们自己翻译用的，并不需要。

### 关于文案 ###
# 文案最好少量多次，单个mod事件能够接受的最大文本量是60行（包括空行），每行最多50字

### Mod Effects ###
# remove_gts_species_size_trait 		移除种族所有体型特质
# gts_species_size_change_to			将种族改为指定体型
# gts_species_size_increase				增大种族体型一个级别
# gts_species_size_reduce				减小种族体型一个级别
# remove_gts_pop_size_trait				移除人口所有体型特质
# gts_pop_size_change_to				将人口改为指定体型
# gts_pop_size_increase					增大人口体型一个级别
# gts_pop_size_reduce					减小人口体型一个级别
# remove_gts_leader_size_trait			一键移除领袖所有体型特质
# gts_leader_size_change_to				将领袖改为指定体型
# gts_leader_size_increase				增大领袖体型一个级别
# gts_leader_size_reduce				减小领袖体型一个级别


### Mod Trigger ###
# is_special_leader						用于判断是否为MOD特殊领袖
# is_eatable_planet						用于判断是否为MOD中不能吃的星球 yes 是有不能吃的 no 是没有不能吃的
# is_gts_leader_planet					用于判断是否为MOD中的巨娘星球
# is_gts_clothes_planet					用于判断是否为MOD中的巨娘衣物
# is_gts_reformed_planet				是GTS改造星球
# gts_can_destory_planet				可以被巨娘事件摧毁的行星
# gts_has_size_trait					拥有体型特质
# gts_is_giantess						是巨娘
# gts_is_tiny							是小人
# gts_is_bigger_than					体型大于$TARGET$
# gts_is_bigger_than_xxx				体型大于特定体型
# gts_is_equal_to						体型等于$TARGET$
# gts_is_equal_to_xxx					体型等于特定体型
# gts_is_smaller_than					体型小于$TARGET$
# gts_is_smaller_than_xxx				体型小于特定体型
# gts_is_tiny_empire					是小人国家
# gts_is_giantess_empire				是巨娘国家
