# ！！！重要，开搞之前请先看一下注意事项！！！
# 1.给你的mod想一个代号，这个代号一定要特殊，最好根据mod的主要内容来确定。
#   mod的代号不要叫submod或者extension，没有特殊性，有导致mod冲突的风险。
#   mod文件的命名格式  gts_mod代号_xxxx
#   mod内事件，flag等变量的命名格式  gts_mod代号_xxxx

# 2.做mod的时候可以修改原版的文本或者贴图，但是尽量不要改原版的代码逻辑

# 3.mod发布前，记得写一个说明文件介绍一下你的mod增加了什么内容
#   说明内容尽量全面一些，方便玩家体验新内容

# ！！！文件命名格式！！！
# 任何文件:避免mod冲突，一律以gts_为前缀，gts_后面可以跟作者名/项目名，加载顺序问题可以再前面加0或者z：000_gts_

# common:如果你的项目包含大量新内容，建议新建一个txt，替换原版内容*必须*单独放在新的txt中，并加上_replace后缀

# events:替换原版事件*必须*单独放在新的txt中，并加上_replace后缀

# gfx/interface/sound:不要重名即可

# localisation: 你的mod围绕什么展开就用什么前缀，比如起源、领袖、国策、事件链
# gts_civ（国策） gts_ap(遗珍) 如果你要围绕一个法令写一堆文案的话可以再开个gts_edict
# gts_o(起源)>gts_i(领袖)>gts_evt(事件链)>gts_com(其他) 前者优先于后者，起源里有领袖的话，文件应该命名为gts_o
# 原版本地化文本中的 :0 一类的数字是给它们自己翻译用的，并不需要。
